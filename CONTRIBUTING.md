# Contributing

Thank you for your interest in contributing to this repository.

This repository is primarily intended as an educational DevOps, Cloud, Kubernetes, Linux, Security, and Platform Engineering knowledge base.

## Ways to Contribute

You can contribute by:

* Fixing typos or formatting issues.
* Improving explanations.
* Updating outdated information.
* Adding examples.
* Adding diagrams.
* Improving documentation structure.
* Reporting inaccuracies.
* Suggesting new topics.

## Before Submitting Changes

Please ensure:

* Content is accurate and technically correct.
* Examples are tested where applicable.
* Markdown formatting is consistent.
* Existing repository structure is maintained.
* Content remains educational and vendor-neutral where possible.

## Pull Requests

When creating a Pull Request:

1. Clearly describe the change.
2. Explain why the change is useful.
3. Keep changes focused and relevant.
4. Reference any related issue if applicable.

## Content Guidelines

Please:

* Use clear and concise language.
* Prefer official documentation as sources.
* Avoid copying copyrighted content.
* Respect third-party licenses and trademarks.
* Provide attribution where appropriate.

## Third-Party Content

Contributors are responsible for ensuring that any content they submit does not violate copyright, trademark, licensing, or intellectual property rights.

By submitting content, contributors confirm that they have the right to contribute that content.

## Disclaimer

All contributions are subject to review and may be modified, accepted, or rejected at the discretion of the repository maintainer.
